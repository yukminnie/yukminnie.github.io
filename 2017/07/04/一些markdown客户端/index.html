一些markdown客户端

Posted on 2017-07-04 Valine:

记录几个用过的 markdown 客户端(不讨论 mou 这 typero 这种纯离线软件)

Cmd Markdown

Cmd Markdown 是刚接触就一直在用的软件.它有很多优点, 全平台,自建服务器，同步稳定,界面美观,导出样式漂亮（可以导出到印象笔记），就我个人来讲，没有特殊需求的时候(比方说自定义图床)，Cmd Markdown 是首选软件。

假如没有上传图片的需求，就不存在收费，很良心的软件。

马克飞象

马克飞象呢，有在线版和谷歌离线应用两种使用方式，按年付费。基本特点和 Cmd Markdown 一致，但是本身并不自带云同步功能，云功能依靠的是印象笔记。

印象笔记之前一直在用，没有广告，免费版每月只有60M 的上传流量，也限制终端数量，假如个人非常依赖印象笔记的话，马克飞象倒是一个很不错的选择。

不过面临马克飞象和印象笔记的双重收费。

MarkEditor

这个软件值得好好说道一下。它的卖点在于：支持文章同步到其自建的服务器上，支持一键发布博客，同时也支持同步到七牛云，亚马逊云。软件本身很精美，也支持一些有意思的功能：黑夜模式，目录生成静态页面，局域网图片导入等。使用过一段时间的感受就是，其自建的云功能不太稳定，指望搜索引擎收录也很困难，很多功能也并不是用户痛点。

说到底就是，其本质上还是个笔记，并没有实现博客该有的功能。

同时，其自建服务也是需要按年收费的。个人觉得，如果存在写博客的需求，使用 Jekyll 和 Hexo 可能会更好。

Nweb

Nweb 和上面提到的 MarkEditor 比较相似。不过 Nweb 并没有提供云功能，那为什么还要提它呢？因为它有个颇为强大的功能：插入图片自动上传图床。

也就是说，文章写好以后，只需要进行复制粘贴，就可以发布到不同的平台，包括你的博客，专栏，微博，微信，笔记文章等等。

这是其最大的卖点，非常适合小编性质的文字工作者。

同时，其支持的静态页面导出功能，和 MarkEditor 有所不同。MarkEditor 仅仅是单纯的导出，Nweb 是支持更新性质的导出，配合 Github pages 可以实现简单的静态博客功能。

Gitbook

支持同步到 Github，支持发布静态文档网站和离线导出。

优点就是好看，缺点就是同步太不稳定，编辑的时候都会卡的样子。

小书匠

没什么说的，该有的都有，功能强大。

丑拒。

Yosoro

一个很纯粹的 MD 笔记软件，编辑时有点小 bug，支持同步到 Onedrive。

Onedrive，恩。

joplin

其实这个应该是我最推荐的，它有个很杀手的功能，就是网页剪藏，因为是本地软件作为服务端，所以剪辑的非常完美，但是同步功能对国内用户不太友好。

如果不需要同步功能，理想的形式是作为笔记本来用，非常推荐这个。

其它

有道云，为知云这些笔记应用，是支持 MD 的。但是比较封闭，并不像印象笔记一样，得到第三方软件广泛的支持。同时个人觉得，其剪辑功能没有印象笔记效果好。

依以往经验来看，国内大厂的软件都不太安分，不纯粹。保不定将来加点社交，推广什么的，这是我不太能接受的。
